Yes, I am very familiar with them! In technical terms, these are often called **"Thumb Buttons"** or **"Side Buttons."**

Most modern mice—whether they are designed for gaming, productivity, or just general office use—include these two extra buttons to help with navigation. 

Here is a quick breakdown of how they usually work and what you can do with them:

### 1. Default Behavior (Out of the Box)
By default, most operating systems (Windows, macOS, Linux) treat Button 4 and Button 5 as navigation shortcuts:
*   **Button 4 (usually the one closer to your wrist):** Acts as the **"Back"** button. In a web browser or a file folder, it takes you back to the previous page or folder.
*   **Button 5 (usually the one further forward):** Acts as the **"Forward"** button. It moves you forward again if you’ve already gone back.

### 2. Customization (Remapping)
The coolest thing about these buttons is that they are highly customizable. If you don't use "Back/Forward" often, you can change them to do almost anything else using software.

*   **Manufacturer Software:** If you have a brand-name mouse (like Logitech, Razer, Corsair, or SteelSeries), they provide software (e.g., Logitech G Hub, Razer Synapse) that lets you assign "Macros" to these buttons.
*   **Third-Party Software:** If you have a generic mouse without official software, there are great tools like **X-Mouse Button Control (Windows)** that let you remap those buttons to things like:
    *   Copy (Ctrl+C) and Paste (Ctrl+V).
    *   Undo (Ctrl+Z).
    *   Muting your microphone during a call.
    *   Opening a specific app.

### 3. Use in Gaming
In gaming, these buttons are incredibly valuable because they allow you to perform actions without moving your fingers off the main movement keys (WASD). Gamers often map them to:
*   Melee attacks.
*   Push-to-Talk for voice chat.
*   Using a specific gadget or potion.

from pynput import mouse, keyboard

kb = keyboard.Controller()

def on_click(x, y, button, pressed):
    if pressed:
        if button == mouse.Button.x1: # Button 4
            # Simulate Ctrl + Left Arrow
            with kb.pressed(keyboard.Key.ctrl):
                kb.press(keyboard.Key.left)
                kb.release(keyboard.Key.left)
        elif button == mouse.Button.x2: # Button 5
            # Simulate Ctrl + Right Arrow
            with kb.pressed(keyboard.Key.ctrl):
                kb.press(keyboard.Key.right)
                kb.release(keyboard.Key.right)

with mouse.Listener(on_click=on_click) as listener:
    listener.join()

### 2. The Code
Replace the contents of `main.swift` with the following code. I have commented it heavily so you can audit exactly what it is doing:

```swift
import Foundation
import CoreGraphics

// --- Configuration ---
// macOS virtual key codes for Arrow keys
let kVK_ArrowLeft: CGKeyCode = 0x7B
let kVK_ArrowRight: CGKeyCode = 0x7C

print("🚀 MouseRemapper started. Listening for Button 4 & 5...")
print("⚠️  Remember to grant Accessibility permissions in System Settings.")

// This is the callback function that processes every event on your system
let callback: CGEventTapCallBack = { (proxy, type, event, refcon) in
    
    // We only care about "Mouse Button Pressed" events
    if type == .mousePressed {
        let buttonNumber = Int(event.getIntegerValueField(.mouseButton))
        
        // Button 3 is typically the 'Back' button (Mouse 4)
        // Button 4 is typically the 'Forward' button (Mouse 5)
        if buttonNumber == 3 || buttonNumber == 4 {
            let keyCode = (buttonNumber == 3) ? kVK_ArrowLeft : kVK_ArrowRight
            print("Detected Button \(buttonNumber), triggering Space Switch...")

            // 1. Create a keyboard event for the Arrow Key
            let keyDown = CGEvent(keyboardEventSource: nil, virtualKey: keyCode, keyDown: true)
            
            // 2. Add the Control key flag (Control + Arrow is the macOS shortcut for Spaces)
            keyDown?.flags = .maskControl
            
            // 3. Create the corresponding Key Up event so the key isn't "stuck" down
            let keyUp = CGEvent(keyboardEventSource: nil, virtualKey: keyCode, keyDown: false)
            keyUp?.flags = .maskControl

            // 4. Post the events to the system
            keyDown?.post(tap: .cgAnnotatedSessionEventTap)
            keyUp?.post(tap: .cgAnnotatedSessionEventTap)

            // Return nil to "swallow" the original event so the browser doesn't go Back/Forward
            return nil 
        }
    }
    
    // For all other events (left click, move, etc.), just pass them through unchanged
    return Unmanaged.passRetained(event)
}

// --- Event Tap Setup ---
let eventMask = (1 &lt;&lt; CGEventType.mousePressed.rawValue)

// Create the tap to listen for mouse presses
guard let eventTap = CGEvent.tapCreate(
    tap: .cgSessionEventTap,
    place: .headInsertEventTap,
    options: .defaultTap,
    eventsOfInterest: CGEventMask(eventMask),
    callback: callback,
    userInfo: nil
) else {
    print("❌ Failed to create event tap. Did you give the app Accessibility permissions?")
    exit(1)
}

// Create a run loop to keep the program alive and listening
let runLoopSource = CFMachPortCreateRunLoopSource(kCFAllocatorDefault, eventTap, 0)
CFRunLoopAddSource(CFRunLoopGetCurrent(), runLoopSource, .commonModes)
CGEvent.tapEnable(tap: eventTap, enable: true)

CFRunLoopRun()
```
